# 0001. 正確性をダメージの条件とし、速度は読み合いに回す

- ステータス: 承認
- 日付: 2026-06-10

## 文脈

マジックタイピングバトルは、リアルタイムのタイピングと TCG 的なデッキ構築を
両立させるゲームである。「タイピングの出来をどうダメージに反映するか」を決める
にあたり、タイピングバトルの一般的な常識は「速く打つほど大ダメージ」である。

しかし本作では、カードごとのお題の長さ・難易度がそのカードの強さに比例する
（強いカード = 長く難しいお題）という構造を採用している。この前提のもとで
「速度 = ダメージ」を重ねると、速度が二重に効いてしまい、正確性の価値が薄れる。

検討した主な代替案:

- **A. 速度がダメージ倍率**: 基本ダメージ × 速度倍率。速く打つほど伸びる。
- **B. 正確性が発動条件、速度はテンポのみ**: 打ち切れば固定ダメージ。
  速度は「相手より先に撃てるか」にのみ効く。
- **C. 速度 × 正確性の合成スコアをダメージ倍率にする**: 管理要素が増える。

## 決定

**B を採用する。** カードのダメージはカードごとに固定とし、速度はダメージに
直接は影響しない。速度が効くのは「相手より先に詠唱を撃ち切れるか」という
読み合いの局面のみである。

誤入力の扱い:

- 誤ったキーは受理されず、正しいキーを打つまで先に進めない（ブロック方式）。
- 加えて、誤入力の回数だけ最終ダメージが減衰する。
- よって誤入力は「時間損失」と「ダメージ減」の二重ペナルティを持つ。

カード経済（強カードに存在意義を与える差別化軸）:

- **クールダウン制**: カード発動後、一定の待ち時間が入り連打を抑制する。
  これにより一回の詠唱枠が貴重になり、その枠を強カードに使う価値が生まれる。
- **非線形リターン**: 強カードはお題長の伸び以上にダメージが伸びる。
  長く難しいお題を正確に打ち切れる実力者ほど報われる。

## 結果

### 良い点

- 「正確に打ち切ること」自体がゲームの中心になり、本作の本質である
  「タイピング実力 × デッキ戦略」に一致する。
- 速度を読み合い（先撃ち）に回すことで、リアルタイム性を保ちつつ
  ダメージ計算をシンプルに保てる。
- 強カード = 長いお題 = 長い詠唱時間 = 相手に先を越されるリスク、という
  自然なハイリスク・ハイリターン構造が生まれる。

### 注意・トレードオフ

- クールダウンが固定時間だと、その待ち時間は強・弱カード共通の死に時間と
  なり、非線形リターンと合わさって強カードが効率でも勝ち、弱カードが死ぬ
  恐れがある。クールダウンをお題長に比例させるか、固定のまま住み分けで
  許容するかは、ベース実装後の実測で決める。
- 誤入力1回あたりの減衰量と下限値もバランス調整事項として未確定。

### この決定がもたらす制約

- 全カードのバランスはこのモデルを前提に組まれるため、後から
  「速度 = ダメージ」へ転換するとデッキ・カード設計をやり直すことになる。

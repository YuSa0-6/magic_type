# 0013. 先行入力(type-ahead)を撤回し、クールダウン中は打鍵を受け付けない

- ステータス: 承認
- 日付: 2026-06-18
- 関連: ADR 0007(本 ADR が撤回する)、ADR 0001(正確性を発動条件としたダメージ)、
  ADR 0008(状態同期の時間軸/入力軸分割)、ADR 0011(対戦ネットコード)。

## 文脈

ADR 0007 でクールダウン中の打鍵を先行入力(type-ahead)としてバッファし、クールダウン
明けにまとめて受理する仕組みを導入した。狙いは、速いプレイヤーがクールダウン明け先頭の
1〜数打鍵を毎発動ごとに落とす「取りこぼし感」の解消だった。

しかし実プレイのフィードバックで、「クールダウンが終わる前にタイピングできてしまう」挙動が
クールダウンという制約の意味を曖昧にし、テンポ設計(クールダウンが詠唱スループットを律速する)
の体感を弱めると判断した。クールダウンは「次の詠唱を始められない待ち時間」であり、その間は
打鍵入力も受け付けない、という素直な挙動に戻したい。

## 決定

**ADR 0007 を撤回する。** 先行入力(type-ahead)機構をシングルプレイ(BattleEngine)と
対戦(PlayerSide / MatchEngine)の両方から完全撤去する。

- クールダウン中も**カード選択(構え)は引き続き可能**。
- クールダウン中の**打鍵は受理せず破棄**する(`pressKey` は `'blocked'` を返す)。
  バッファも繰り越しも無い(ADR 0007 導入前の挙動に戻す)。
- `PressResult` から `'buffered'` を削除。`drainTypeahead` メソッド・`typeahead` バッファ・
  上限(`TYPEAHEAD_LIMIT`)・直列化 DTO の `typeahead` フィールドを廃止。
- 対戦のコマンドログから `'drain'` コマンドが消え、リプレイ(`fromCommands`, ADR 0011 #4)は
  start / select / press のみの純粋な入力駆動になる。

### 決定論への影響(対戦)

ラグ補償(`MatchSession` の `authClock` リプレイ, ADR 0011)は**維持**する。今回外すのは
per-side の type-ahead バッファだけである。クールダウン境界(`cooldownUntilMs`)は権威的な
発動 `atMs` から一意に決まるため、「クールダウン中の打鍵を破棄」してもクライアント予測と
サーバー権威は同じ判定になり、決定論は保たれる。むしろドレイン契機(時間 tick 駆動)が
コマンドログから消えることで、リプレイは入力列のみで完全に再現でき、決定論がより明確になる。

## 結果

### 良い点

- クールダウンの制約が明快になり、テンポ設計の体感が戻る。
- 対戦の権威リプレイが start / select / press のみの純粋な入力駆動になり、ドレインの
  tick 依存が消えて単純化・決定論が明確化する。
- エンジンから typeahead バッファ・上限・リセット規則・DTO フィールドが消え、状態が縮小する。

### トレードオフ

- ADR 0007 が解消した「クールダウン明け先頭打鍵の取りこぼし感」は再び表面化する
  (上級者はクールダウン明けの瞬間を待ってから詠唱を始める必要がある)。これは
  クールダウンの意味を優先する意図的なゲーム性の選択として受け入れる。
- クールダウン中に詠唱を先行入力で進めてスタックしておくことはできない。

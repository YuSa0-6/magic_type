# CONTEXT.md — ユビキタス言語

このプロジェクトで使う用語の定義。実装詳細は書かない。

## ゲームの基本

| 用語           | 英語名               | 定義                                                                                                                                                                                                                                                                                                                                                                                   |
| -------------- | -------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| お題           | Prompt               | プレイヤーに1回表示され、打ち切る単位。**表示テキスト**(漢字かな交じり文)と**読み**(かな文字列)のペア。各カードは固定のお題を1つ持つ。お題の長さはそのカードの詠唱時間(=先撃ちで晒すリスク)を決め、1発の絶対ダメージ(カードダメージ)は概ねお題長に比例する。ただしクイックカードはこの比例の例外で、短い詠唱とテンポ価値と引き換えに絶対ダメージを切り詰める。                                                                                                                |
| 読み           | Reading              | お題の正となる文字列。プレイヤーはこれを再現するよう打つ。表示テキストではなく読みが判定の正である。中身と照合の仕方は照合モードで決まる(かな照合では読みはかな文字列、リテラル照合では読みはASCII文字列そのもので読み≒表示テキストになる)。                                                                                                                                                                                                                                                                           |
| 照合モード | Matching Mode | 各カードのお題が「読み」をどう照合するかの規則。**かな照合**(既定)では読みはかな文字列で、ローマ字入力を表記ゆれ込みで照合する。**リテラル照合**では読みはASCII文字列で、1キー=1文字を確定的に照合する(表記ゆれなし)。どの照合モードでも「読みが判定の正」「誤入力数がダメージを減衰させる」という不変条件は共通。 |
| 表記ゆれ       | Romanization Variant | (かな照合モードでのみ生じる)同じ読みに対して有効な複数のローマ字入力(例: し=si/shi/ci、ん=n/nn、っ=子音重ね/xtu/ltu)。判定エンジンはすべての有効な表記を正解として受理する。リテラル照合モードには表記ゆれはない。                                                                                                                                                                                                                                       |
| 詠唱           | Casting              | カードのお題を打ち切り、効果を発動させる行為。詠唱が完了するとカード発動が起こる。                                                                                                                                                                                                                                                                                                     |
| ローマ字ガイド | Romaji Guide         | 詠唱中に表示される、残りの推奨ローマ字列。プレイヤーの打鍵に追従して動的に更新される(例: 「し」に対して s の後に h を打てば shi ルートで残りが再表示される)。表記ゆれ全受理と表示を矛盾させないための仕組み。                                                                                                                                                                          |
| 誤入力         | Mistype              | 詠唱中に正しくないキーを打つこと。「正しい」とは、残りの読みのいずれかの有効な表記(表記ゆれ含む)の継続キーであることを指す。途中まで打ったどの流派の続きとしても成立しないキーが誤入力となる。誤ったキーは受理されず、正しいキーを打つまで先に進めない(ブロック)。加えて誤入力の回数はそのカードの最終ダメージを減衰させる。よって誤入力は時間損失とダメージ減の二重ペナルティを持つ。 |

## カードの強さ

「強い／弱い」は3つの別々の尺度を指しうる。混同しないために分けて定義する。

| 用語           | 英語名                   | 定義                                                                                                                                                                                                                                                             |
| -------------- | ------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| カードダメージ | Card Damage              | カード1発の固定ダメージ。誤入力数だけ減衰する(下限1)。打鍵速度には依存しない(ADR 0001)。「強い／弱い」を1発の絶対量で語るときはこの尺度を指す。                                                                                                                   |
| 打鍵効率       | Per-Keystroke Efficiency | カードダメージを、そのお題の最短打鍵数で割った値。1打鍵に投じた労力あたりのダメージ。通常お題は長いものほど効率が高い「非線形リターン」を持つ(同じ打鍵数を費やすなら長いお題1枚が有利)。                                                                           |
| 実効DPS        | Effective DPS            | 単位時間あたりのダメージ。1発のダメージを cycle(=詠唱時間とクールダウンの大きい方)で割った値。試合のテンポ(押し引きの速さ)を決める尺度。打鍵効率が高くても、絶対ダメージが小さくクールダウンが律速になれば実効DPSは伸びない。                                       |
| クイックカード | Quick Card               | 詠唱が非常に短いカード群。絶対ダメージ(カードダメージ)は小さいが打鍵効率は高い。短い詠唱ゆえにクールダウンが律速になり、テンポ軸(クールダウンの伸縮)が初めて効く局面を作ること自体がそのカードの価値であって、絶対ダメージではない。実効DPSが暴れないよう設計で抑える。 |

## モード

| 用語           | 英語名      | 定義                                                                                                                                                       |
| -------------- | ----------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------- |
| ソロモード     | Solo Mode   | 練習用の一人プレイ。オフラインで完結する。                                                                                                                 |
| タイムアタック | Time Attack | ソロモードの一形態。行動しないサンドバッグの的(一定HP)を倒すまでの時間を競う。相手の行動を持たず、コアループとタイピングの手触りを検証するための最小構成。 |
| 対戦モード     | Versus Mode | オンラインで他のプレイヤーと同時にプレイし勝敗を決めるモード。                                                                                             |

## デッキ

| 用語   | 英語名       | 定義                                                                                                   |
| ------ | ------------ | ------------------------------------------------------------------------------------------------------ |
| デッキ | Deck         | プレイヤーが事前に構築する15枚のカードの集合。同じカードは最大2枚まで。                                |
| 山札   | Draw Pile    | バトル中、まだ引かれていないカードの束。バトル開始時にデッキをシャッフルして作る。                     |
| 捨て札 | Discard Pile | 発動済みのカードが溜まる置き場。山札が尽きたら捨て札をシャッフルして新しい山札にする(デッキのループ)。 |

## バトル操作

| 用語         | 英語名          | 定義                                                                                                                                                                                                                                   |
| ------------ | --------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| カード選択   | Card Selection  | 手札の中から使用するカードを指定する操作。キーボードショートカット（1〜4キー）またはマウスクリックで行う。選択後にそのカードのお題が表示されてタイピングが開始される。タイピング中に別のカードに切り替えた場合、進捗はリセットされる。 |
| 手札         | Hand            | プレイヤーが現在使用可能なカードの集合。常に4枚を維持する。カード発動と同時にデッキから1枚補充される。                                                                                                                                 |
| カード発動   | Card Activation | お題を打ち切ることでカードの効果が実行される瞬間。                                                                                                                                                                                     |
| クールダウン | Cooldown        | カード発動の直後、次のカードを詠唱開始できない待ち時間。連打を抑制し、一回の詠唱枠の価値を生む。クールダウン中もカード選択(構え)は可能で、構え済みなら打鍵を先行入力として受け付けるが、受理が確定するのはクールダウン明けである。                                 |
| 先行入力     | Type-ahead      | クールダウン中に構え済みのカードへ打った打鍵をバッファし、クールダウン明けにまとめて受理する仕組み。クールダウン明けの瞬間を目視で待たずに詠唱へ移れる。受理時刻はクールダウン明けで統一するため、先行入力しても詠唱時間は不当に伸びない。別カードへ構え直すとバッファは進捗ごとリセットされる。 |

## 対戦

| 用語       | 英語名   | 定義                                                                                                                                                                                                                                                                                                                                          |
| ---------- | -------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| マッチ     | Match    | 対戦モードで2人のプレイヤーが1回戦い、勝敗を決する単位。両者がHPを持ち、先に相手のHPを0にした側が勝つ。決着しないまま制限時間が尽きた場合は残HPの多い側が勝つ。                                                                                                                                                                                  |
| 対戦相手   | Opponent | 対戦モードで自分と勝敗を争う、もう一方のプレイヤー。エンジン上は自陣と同型の状態(HP・詠唱・クールダウン等)で表現される。ソロモードのサンドバッグの的は、行動しない対戦相手として同型で表す(ADR 0009)。                                                                                                                                            |
| 効果       | Effect   | カード発動時に、固定ダメージに加えて発生するゲーム状態への作用。**相手のゲーム経済**(HP・クールダウン・手札・山札等)への妨害と、**自陣**への強化(バフ)の二系統がある。相手の打鍵そのもの(入力・ガイド表示・誤入力)には一切介入しない。これによりサーバー権威での決定論的検証が可能になり、本作の核「タイピング実力 × デッキ戦略」を薄めない。 |
| 妨害       | Debuff   | 効果のうち、相手のゲーム経済に不利を与えるもの(例: 相手のクールダウン延長、相手の手札をランダムに捨てさせる)。相手の打鍵体験には触れない。                                                                                                                                                                                                     |
| 強化       | Buff     | 効果のうち、自陣に有利を与えるもの(例: シールド・回復・自分のクールダウン短縮・自分の山札操作)。                                                                                                                                                                                                                                               |
